# Rule: Coverage Policy

**Priority**: P0 — Workflow не завершается, пока coverage < 70%.

1. **Минимальный порог**: 70% line coverage по `src/` указанного проекта.
2. **Итеративный цикл обязателен**: После каждого запуска анализировать отчёт coverage, определять непокрытые строки и ДОПИСЫВАТЬ тесты. Нельзя остановиться после первого прогона.
3. **Считается только source coverage**: Coverage измеряется по `src/`, не по самим тестам.
4. **Оба типа сценариев обязательны**: Нельзя закрыть 70% только позитивными или только негативными тестами.
5. **Реальное выполнение**: Все тесты должны пройти (pytest exit code 0) перед финальным отчётом.
